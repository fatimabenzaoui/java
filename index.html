Concepts POO Java
CONCEPTS SENS
CLASSE Une classe est un bloc de code contenant des propriétés, un constructeur, des méthodes et qui sert de plan pour créer des objets. Pour nommer une classe, on utilise le Pascal Case.
Le Pascal Case est une convention d'écriture : les mots sont liés sans espace et chaque mot commence par une lettre capitale.
Une propriété est une variable ou une constante définie à l’intérieur d’une classe.
Il existe deux types de variables :
- les variables d'instance qui appartiennent uniquement à l'objet
- les variables de classe qui appartiennent à la classe et sont partagées par tous les objets (static)
Une constante est un espace réservé en mémoire qui contient une valeur dont le contenu ne peut être modifié.
Le rôle principal d'un constructeur consiste à initialiser les propriétés.
Une méthode est une fonction définie à l’intérieur d’une classe. Elle possède une signature et un corps, peut posséder des paramètres et renvoyer une valeur de retour.
OBJET Un objet est une variable stockant une instance de classe. Un objet a ainsi accès aux propriétés et méthodes des classes dont il est issu.
ENCAPSULATION L'encapsulation correspond à la façon dont on définit la visibilité et l’accessibilité des propriétés et méthodes des classes. L'enjeu étant de cacher le plus d'informations possible.
"Public" indique que la propriété ou la méthode est accessible à l’extérieur de la classe.
"Private" indique que la propriété ou la méthode est accessible uniquement depuis l’intérieur de la classe.
"Protected" indique que la propriété ou la méthode est accessible dans la classe mère ainsi que dans les classes qui l’étendent mais aussi dans les classes qui sont dans le même package.
Un getter est une méthode qui sert à récupérer une valeur.
Un setter est une méthode qui sert à définir une valeur.
HERITAGE On parle d'héritage car les classes filles héritent des méthodes et propriétés de leur mère mais également de celles de leur grand-mère etc... La relation d'héritage décrit une relation de type : "est un", "est un type de".
Le polymorphisme est l'aptitude d'un objet à pouvoir prendre plusieurs formes : en effet, tout objet d'une classe fille est aussi considéré comme un objet des classes mères (ex : Animal rex = new Chien();).
STATIC Une classe dite static est une classe qui ne contient que des attributs et des méthodes static. Celle-ci n'a donc pas à être instanciée (ex: Math, Console).
Une propriété dite static est une variable de classe spécifique à la classe et partagée par tous les objets (# variable d'instance qui appartient à l'objet).
Une méthode dite static est une méthode qui n'a pas besoin que la classe soit instanciée et qu'un nouvel objet soit créé pour être utilisée. Une méthode static est commune à tt le monde et est partagée par tt le monde. Certaines méthodes ont besoin d'être accessibles même sans objet (ex : main()). Elles sont dites static car elles sont liées à la classe et non à un objet.
FINAL Une classe, une constante ou une méthode déclarée final ne peut pas être modifiée : sa redéfinition grâce à l'héritage est interdite.
ABSTRACT Une classe abstraite est une classe qu’on ne peut pas instancier (pas de création d’objet).
Une méthode abstraite est une méthode qui n’a pas de corps et qui doit obligatoirement être définie dans toutes les classes filles.
INTERFACE Une interface est un contrat que les classes vont devoir suivre : elle n'est composée que de méthodes abstraites (signatures de méthodes) qui doivent être définies par les classes qui l'implémente. L'avantage est que l'on va pouvoir implémenter autant d'interfaces que l'on souhaite (# classe abstraite : on ne peut hériter que d'une seule classe).
